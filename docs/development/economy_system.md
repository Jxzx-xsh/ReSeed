# 经济与资源系统设计

> 为种子城（Seed City）定义货币体系、商品分类、价格波动机制，以及玩家交易对世界的影响。

---

## 一、货币体系：信用点（Credit）

### 1.1 设计理念

种子城没有传统货币。"信用点"是一种**社会贡献积分**，由锈蚀议会根据居民对城市的贡献自动计算。它不是实物，而是记录在铁砧维护的中央账本（一台旧服务器）上的数字。

### 1.2 信用点获取方式

| 行为 | 获得信用点 | 说明 |
|------|-----------|------|
| 完成拾荒任务 | 5~30 | 取决于带回物资的稀缺度 |
| 净水站值班 | 3/天 | 基础劳动报酬 |
| 温室种植协助 | 2~8 | 根据产出量计算 |
| 信使传递 | 10~50 | 距离越远、风险越高，报酬越多 |
| 维修公共设施 | 5~20 | 铁砧评估工作量后发放 |
| 提供有效情报 | 5~40 | 苏漫或议会验证后发放 |
| 击退灰灵/威胁 | 20~100 | 视威胁等级而定 |

### 1.3 信用点消耗

- 购买商品（见下方商品表）
- 租用工具/装备
- 请求NPC协助（如请铁砧修理装备）
- 进入特殊区域（如回声井需要"通行许可"）
- 黑冰市场的非官方交易（苏漫接受信用点，但汇率浮动）

### 1.4 经济规则

- **每日基础配给**：每位居民免费获得 2L 净水 + 1份基础口粮（不消耗信用点）
- **超额消费**：超出配给的部分需要信用点购买
- **负信用**：信用点可以为负（欠债），但低于 -50 时会被议会限制行动自由
- **通胀控制**：议会每季度调整一次信用点发放标准

## 三、价格波动机制

### 3.1 供需模型

价格围绕基础价格浮动，受以下因素影响：

```
实际价格 = 基础价格 × 供给系数 × 需求系数 × 季节系数 × 事件系数
```

### 3.2 供给系数（0.5 ~ 2.0）

| 条件 | 系数 | 触发 |
|------|------|------|
| 供给充足 | 0.5~0.8 | 拾荒大丰收、温室高产期 |
| 供给正常 | 1.0 | 默认状态 |
| 供给紧张 | 1.3~1.5 | 拾荒队损失、设备故障 |
| 供给断裂 | 1.8~2.0 | 锈蚀风暴封路、灰灵袭击 |

### 3.3 需求系数（0.7 ~ 2.5）

| 条件 | 系数 | 触发 |
|------|------|------|
| 需求低迷 | 0.7~0.9 | 人口减少、替代品出现 |
| 需求正常 | 1.0 | 默认状态 |
| 需求旺盛 | 1.3~1.8 | 新居民涌入、季节转换 |
| 恐慌抢购 | 2.0~2.5 | 灰灵入侵预警、风暴来临 |

### 3.4 季节系数

| 季节 | 水 | 食物 | 零件 | 药品 |
|------|-----|------|------|------|
| 灰季 | ×1.3 | ×1.0 | ×0.9 | ×1.2 |
| 酸雨季 | ×0.8 | ×1.5 | ×1.4 | ×1.8 |
| 苏醒季 | ×1.0 | ×0.7 | ×1.0 | ×0.9 |

**说明**：
- 灰季干燥缺水，水价上涨
- 酸雨季食物易腐、零件被腐蚀、感染风险高
- 苏醒季植物丰收，食物和药品价格下降

### 3.5 事件系数（动态触发）

| 事件 | 影响商品 | 系数 | 持续时间 |
|------|---------|------|---------|
| 锈蚀风暴 | 零件 ↑、药品 ↑ | ×1.5 | 3~5天 |
| 灰灵入侵 | 全品类 ↑ | ×1.3 | 事件结束前 |
| 温室故障 | 食物 ↑ | ×2.0 | 修复前 |
| 净水站停摆 | 水 ↑ | ×3.0 | 修复前 |
| 拾荒大发现 | 对应品类 ↓ | ×0.5 | 7天 |
| 新贸易路线开通 | 全品类 ↓ | ×0.8 | 永久 |

## 四、交易场所与规则

### 4.1 官方交易：中心广场

- 由锈蚀议会监管
- 价格透明，按公示牌标价
- 每日更新价格（铁砧计算）
- 禁止违禁品交易

### 4.2 黑市交易：黑冰市场（苏漫）

- 无监管，价格由苏漫决定
- 可交易违禁品
- 接受以物易物（苏漫自定汇率）
- 情报交易的主要场所
- **苏漫加价规则**：基础价格 ×1.2~1.8（取决于她对买家的评估）

### 4.3 私人交易

- 玩家可与任何NPC直接交易
- 不受官方价格约束
- 需要建立足够的信任度
- 可能触发特殊任务线

## 五、玩家影响世界经济的方式

### 5.1 直接影响

| 玩家行为 | 经济影响 |
|---------|---------|
| 大量出售某商品 | 该商品供给系数下降（价格降低） |
| 大量购买/囤积 | 该商品需求系数上升（价格上涨） |
| 完成拾荒任务 | 增加对应资源供给 |
| 修复净水站/温室 | 永久降低对应商品基础价格 |
| 开辟新贸易路线 | 全局价格下降 |
| 破坏基础设施 | 对应商品价格飙升 |

### 5.2 间接影响

| 玩家行为 | 连锁反应 |
|---------|---------|
| 帮助阿洛培育超级苔藓 | 食物基础价格永久 -20% |
| 帮沈沫修复纳米过滤机 | 水基础价格永久 -30%，纳米感染率下降 |
| 暴露苏漫与灰灵的交易 | 黑冰市场关闭1周，之后价格 +50%（信任危机） |
| 给低语者提供算力 | 获得旧地图 → 发现新资源点 → 供给增加 |
| 唤醒沉睡者 | 可能获得制造蓝图 → 新商品出现 |

### 5.3 经济声望系统

玩家的交易行为会影响NPC对其的态度：

| 声望等级 | 条件 | 效果 |
|---------|------|------|
| 信赖 | 多次公平交易，完成任务 | 价格 -10%，解锁特殊商品 |
| 中立 | 默认状态 | 标准价格 |
| 警惕 | 囤积居奇、哄抬物价 | 价格 +20%，部分NPC拒绝交易 |
| 敌对 | 破坏经济、与灰灵勾结 | 被黑冰市场禁入，议会限制配给 |

## 六、技术实现备注

### 6.1 数据结构（参考）

```typescript
interface Commodity {
  id: string;
  name: string;
  category: 'survival' | 'tools' | 'info' | 'contraband';
  basePrice: number;
  currentPrice: number;
  supply: number;        // 当前库存
  maxSupply: number;     // 最大库存容量
  demandRate: number;    // 每tick消耗量
  restockRate: number;   // 每tick补充量
}

interface PriceModifier {
  type: 'supply' | 'demand' | 'season' | 'event';
  multiplier: number;
  duration: number;      // -1 为永久
  source: string;        // 触发来源
}

interface Transaction {
  buyer: string;
  seller: string;
  commodity: string;
  quantity: number;
  pricePerUnit: number;
  timestamp: number;
  location: 'plaza' | 'blackmarket' | 'private';
}
```

### 6.2 Tick 更新逻辑

每个游戏 Tick（约 1 游戏小时）：
1. 消耗资源（居民日常需求）
2. 补充资源（温室产出、净水站产出）
3. 重新计算价格系数
4. 检查事件触发条件
5. 更新NPC交易意愿

### 6.3 与现有系统的接口

- **世界状态（server/src/world/）**：季节、事件、Tick 推进
- **NPC代理（server/src/agents/）**：交易意愿、声望判断
- **WebSocket（server/src/websocket/）**：价格更新推送给客户端
- **UI（client/src/ui/）**：交易界面、价格公示牌
